# AI Agent Capabilities

## Overview

This document describes the capabilities available when working with the Azure App Configuration Kubernetes Provider repository through AI-powered development agents.

## Agent Skills

### Code Understanding
- **Repository Analysis**: Ability to explore and understand the codebase structure
- **Dependency Management**: Understanding Go modules and dependencies
- **Architecture Comprehension**: Grasp of Kubernetes operator patterns and Azure SDK usage

### Code Modification
- **Precise Edits**: Making surgical, minimal changes to existing code
- **File Creation**: Creating new files following project conventions
- **Refactoring**: Restructuring code while maintaining functionality

### Testing & Validation
- **Test Execution**: Running existing test suites (Go tests)
- **Build Verification**: Executing build commands and validating output
- **Linting**: Running code quality checks using configured linters

### Documentation
- **Documentation Updates**: Maintaining README, API documentation, and code comments
- **Change Documentation**: Documenting changes and their rationale

### Version Control
- **Git Operations**: Committing changes with meaningful messages
- **Branch Management**: Working on feature branches
- **Progress Reporting**: Regular updates on task completion

### Azure & Kubernetes Expertise
- **Azure App Configuration**: Understanding of Azure App Configuration service
- **Kubernetes Operators**: Knowledge of Kubernetes custom resources and controllers
- **Go Development**: Proficiency in Go programming language

## Automated Capabilities

The AI agent can execute the following tasks autonomously:

1. **Code Search**: Fast pattern matching across the codebase

3. **Command Execution**: Running shell commands for building, testing, and deployment

5. **Security Scanning**: Checking for vulnerabilities in code and dependencies
